# Data Critique & Recommendation Template

## 1. Executive Summary
- Purpose and context of the data critique
- Key findings and recommendations

## 2. Evaluation Criteria
- Data quality (completeness, accuracy, consistency)
- Data lineage and provenance
- Data privacy and compliance
- Data accessibility and usability

## 3. Current Data Assessment
### 3.1 Strengths
- Notable data assets, practices, or quality

### 3.2 Weaknesses / Limitations
- Data gaps, quality issues, or risks
- Evidence/examples from profiling or incidents

### 3.3 Gaps vs. Requirements
- Comparison to data requirements/specifications
- Unmet needs or misalignments

## 4. Alternative Approaches Considered
- Other data sources, formats, or pipelines
- Pros and cons of each

## 5. Recommendations
- Actionable improvements for data quality, governance, or architecture
- Prioritization and impact analysis

## 6. Implementation Guidance
- Steps for data remediation or enhancement
- Roles and responsibilities

## 7. Appendices
- Data profiling reports, lineage diagrams, references
